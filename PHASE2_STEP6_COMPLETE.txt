✅ 2. API ENDPOINTS
     File: main.py (lines 1305-1406)
     
     ◆ POST /search
       Request: { query, top_k? }
       Response: { query, results[], total_results, search_time_ms, highest_score }
       Authentication: Bearer token required
       User isolation: Automatic filtering by user_id
     
     ◆ POST /documents/{id}/search
       Request: { query, top_k? }
       Response: Same as /search
       Scope: Single document only
       Ownership: Verified before search
     
     ◆ GET /search-statistics
       Response: { documents_count, chunks_count, embeddings_count, searchable }
       Purpose: Display search availability

Global Search Endpoint
───────────────────────────────────────────────────────────────────────────────

Endpoint:    POST /search
Auth:        Bearer {JWT_TOKEN}
Content:     application/json

Request:
{
  "query": "effects of microgravity on plants",
  "top_k": 5
}

Response (200 OK):
{
  "query": "effects of microgravity on plants",
  "results": [
    {
      "document_id": 1,
      "chunk_id": 34,
      "filename": "space_plants.pdf",
      "chunk_text": "Plants grown in microgravity show significant changes...",
      "similarity_score": 0.91,
      "chunk_index": 33
    },
    {
      "document_id": 2,
      "chunk_id": 89,
      "filename": "biology_research.pdf",
      "chunk_text": "Microgravity has profound effects on plant development...",
      "similarity_score": 0.87,
      "chunk_index": 88
    }
  ],
  "total_results": 2,
  "search_time_ms": 87.3,
  "highest_similarity_score": 0.91
}

Error Response (400):
{
  "detail": "Search query cannot be empty."
}

Error Response (503):
{
  "detail": "Vector store query failed. Please check that your documents have embeddings."
}

───────────────────────────────────────────────────────────────────────────────

Document-Specific Search
───────────────────────────────────────────────────────────────────────────────

Endpoint:    POST /documents/{document_id}/search
Auth:        Bearer {JWT_TOKEN}
Params:      document_id (path parameter)
Content:     application/json

Request:
{
  "query": "cellular response",
  "top_k": 3
}

Response: Same schema as global search
          (filtered to selected document only)

Note: Document ownership verified before search
      Error 400 if document not found or not owned

Search Statistics
───────────────────────────────────────────────────────────────────────────────

Endpoint:    GET /search-statistics
Auth:        Bearer {JWT_TOKEN}

Response (200 OK):
{
  "documents_count": 5,
  "chunks_count": 150,
  "embeddings_count": 150,
  "searchable": true
}

Use: To display search availability in UI

User Input
    │
    ├─→ "plant growth in space"
    │
    ↓
SearchPanel Component
    │
    ├─→ Query validation
    ├─→ Top K selection (1-20)
    │
    ↓
performSemanticSearch() API Call
    │
    ├─→ POST /search
    ├─→ Bearer {JWT Token}
    │
    ↓
Backend: main.py endpoint
    │
    ├─→ get_current_user() ← JWT extraction
    │
    ↓
services/semantic_search.py
    │
    ├─→ validate_search_query()
    ├─→ validate_top_k()
    ├─→ get_embedding_model()
    ├─→ generate_embeddings([query])
    │
    ↓
ChromaDB Query
    │
    ├─→ Query vector: [0.1, -0.2, 0.3, ...]
    ├─→ Filter: {user_id: 42}
    ├─→ Top K: 5
    │
    ↓
Vector Similarity Search
    │
    ├─→ Cosine similarity scoring
    ├─→ Result ranking (descending score)
    │
    ↓
Format Response
    │
    ├─→ Convert distances → similarities
    ├─→ Add metadata (filename, chunk_index)
    ├─→ Calculate execution time
    │
    ↓
JSON Response
    │
    ├─→ {query, results[], analytics}
    │
    ↓
SearchResultsModal
    │
    ├─→ Display results with scores
    ├─→ Show analytics (time, count, best)
    ├─→ Error handling if needed

ChromaDB Collection: knowledge_documents

Metadata per Vector:
{
  "document_id": 1,        ← Used for filtering
  "chunk_id": 34,          ← Identifies chunk
  "chunk_index": 33,       ← Position in document
  "filename": "doc.pdf",   ← Display
  "user_id": 42            ← USER ISOLATION KEY
}

Query Filter:
{
  "user_id": int(current_user.id)  ← Ensures user sees only own docs
}

Optional Additional Filter (for document search):
{
  "user_id": int(current_user.id),
  "document_id": int(document_id)
}
